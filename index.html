1. Clases y Objetos: Los Planos y las Casas

Imagina que quieres construir una casa. Primero, necesitas un plano que defina cómo será. En POO, ese plano es una clase. Una vez que tienes el plano, puedes construir una o muchas casas a partir de él. Cada casa es una instancia u objeto de esa clase.

Clase (El Plano)

Ejemplo en Código:

class Carro:
    pass

# Creamos dos objetos (dos instancias) a partir de la clase Carro
mi_carro = Carro()
otro_carro = Carro()

print(mi_carro)
print(otro_carro)

2. El Constructor (`__init__`)

Cuando creamos un objeto, a menudo queremos darle un estado inicial. Para eso usamos un método especial llamado __init__, que es el constructor de la clase. Se ejecuta automáticamente cada vez que se crea un nuevo objeto.

Ejemplo en Código:

class Carro:
    def __init__(self, marca, modelo):
        # Estas son las propiedades o atributos
        self.marca = marca
        self.modelo = modelo

mi_mazda = Carro("Mazda", "3")
print(f"Tengo un {mi_mazda.marca} modelo {mi_mazda.modelo}")

Sobrecarga de Constructores

Python no soporta la sobrecarga de constructores como otros lenguajes, pero puedes lograr un efecto similar usando parámetros con valores por defecto.

class Persona:
    def __init__(self, nombre, edad=0): # edad tiene un valor por defecto
        self.nombre = nombre
        self.edad = edad

adulto = Persona("Juan", 30)
niño = Persona("Pedrito") # La edad será 0

3. Propiedades y Métodos

  • Propiedades (o atributos): Son las características o datos que pertenecen a un objeto (como `marca` y `modelo`).
  • Métodos: Son las acciones o comportamientos que un objeto puede realizar (como `encender()` o `frenar()`).
class Carro:
    def __init__(self, marca, modelo):
        self.marca = marca
        self.modelo = modelo
        self.encendido = False

    # Método para encender el carro
    def encender(self):
        self.encendido = True
        print("El carro ha sido encendido.")

mi_carro = Carro("Toyota", "Prado")
mi_carro.encender()

4. Encapsulamiento

Es la idea de ocultar los detalles internos de un objeto y exponer solo lo que es necesario. En Python, se usan convenciones con guiones bajos para indicar la visibilidad:

_protegido (Un guion bajo)

Indica que no debería ser modificado fuera de la clase o sus subclases. Es una advertencia.

__privado (Doble guion bajo)

Indica que es "privado". Python cambia el nombre para hacerlo más difícil de acceder desde fuera.

class CuentaBancaria:
    def __init__(self, saldo_inicial):
        self.__saldo = saldo_inicial # Atributo "privado"

    def ver_saldo(self):
        return self.__saldo

mi_cuenta = CuentaBancaria(1000)
print(mi_cuenta.ver_saldo()) # Forma correcta de acceder
# print(mi_cuenta.__saldo) # Esto dará un error

5. Herencia

La herencia permite que una clase (hija) herede las propiedades y métodos de otra clase (padre). Esto es genial para reutilizar código y crear jerarquías lógicas.

Animal (Clase Padre)

Perro (Clase Hija)

Gato (Clase Hija)

class Animal:
    def hacer_sonido(self):
        print("Sonido genérico")

class Perro(Animal): # Perro hereda de Animal
    def hacer_sonido(self): # Sobrescribe el método del padre
        print("¡Guau!")

mi_perro = Perro()
mi_perro.hacer_sonido() # Imprimirá "¡Guau!"

6. Otros Conceptos Importantes

Métodos Estáticos

Un método que pertenece a la clase, pero no depende de ninguna instancia. Se definen con @staticmethod.

class Calculadora:
    @staticmethod
    def sumar(a, b):
        return a + b

resultado = Calculadora.sumar(5, 10)

Método `__str__` (toString)

Define una representación legible para un objeto. Es lo que se muestra cuando usas print().

class Libro:
    def __init__(self, titulo):
        self.titulo = titulo

    def __str__(self):
        return self.titulo

mi_libro = Libro("El Quijote")
print(mi_libro) # Imprime "El Quijote"
