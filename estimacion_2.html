Estimación por Puntos de Historia y Velocidad

Objetivo: Aprender a estimar con puntos de historia, calcular la velocidad del equipo y proyectar fechas de entrega.

Introducción a Scrum

Scrum es un marco ágil para gestionar proyectos, especialmente en desarrollo de software. Promueve la colaboración, la entrega incremental y la adaptación al cambio.

  • Roles:
    • Product Owner: Define la visión del producto y prioriza el backlog.
    • Scrum Master: Facilita el proceso, elimina impedimentos y asegura que el equipo siga las prácticas de Scrum.
    • Equipo de Desarrollo: Ejecuta el trabajo, autoorganizado y multidisciplinario.
  • Eventos:
    • Sprint: Iteración de 1-4 semanas para entregar un incremento funcional.
    • Daily Scrum: Reunión diaria de 15 minutos para sincronizar el equipo.
    • Sprint Planning: Planificación de las tareas del sprint.
    • Sprint Review: Demostración del incremento al final del sprint.
    • Sprint Retrospective: Reflexión para mejorar procesos.
  • Artefactos:
    • Product Backlog: Lista priorizada de tareas o historias de usuario.
    • Sprint Backlog: Subconjunto del backlog seleccionado para el sprint.
    • Incremento: Producto funcional entregado al final del sprint.

Conceptos Clave

1. Puntos de Historia (Story Points)

Los puntos de historia miden el esfuerzo relativo para completar una historia de usuario, considerando complejidad, esfuerzo y riesgo.

  • Esfuerzo Relativo: Compara tareas (e.g., una tarea de 5 puntos es más compleja que una de 1 punto).
  • Escala de Fibonacci: Usa valores como 1, 2, 3, 5, 8, 13 para reflejar incertidumbre.
  • Definición de "Hecho" (Definition of Done): Criterios para considerar una historia completa (e.g., codificado, probado, desplegado).

2. Velocidad del Equipo

La velocidad es el promedio de puntos de historia completados por sprint. Ayuda a planificar y proyectar entregas.

  • Cálculo: Promedio de puntos completados en sprints recientes.
  • Proyección: Estima sprints necesarios para el backlog.
  • Variabilidad: Puede fluctuar por cambios en el equipo o complejidad.

3. Burndown Chart

Un Burndown Chart es un gráfico que muestra el progreso del equipo al completar puntos de historia durante un sprint o proyecto. Visualiza los puntos restantes frente al tiempo, ayudando a identificar si el equipo está en camino de cumplir los objetivos.

  • Eje Y: Puntos de historia restantes.
  • Eje X: Tiempo (días o sprints).
  • Línea Ideal: Representa el ritmo esperado si el trabajo se completa uniformemente.
  • Línea Real: Muestra el progreso actual, permitiendo detectar desviaciones.

4. Terminología Complementaria

  • Product Backlog: Lista priorizada de tareas o historias de usuario.
  • Sprint Backlog: Tareas seleccionadas para un sprint.
  • Planning Poker: Técnica colaborativa para asignar puntos de historia.
  • Incremento: Producto funcional entregado al final del sprint.

Actividad Práctica: Simulación de Velocidad y Proyección

Usa Python para simular la velocidad de un equipo y proyectar fechas de entrega. Genera un Burndown Chart con matplotlib.

Paso 1: Configuración del Entorno

Crea un entorno virtual e instala las librerías necesarias:

mkdir simulacion_velocidad
cd simulacion_velocidad
python -m venv venv
# Activar (Windows):
.\venv\Scripts\activate
# Activar (Mac/Linux):
source venv/bin/activate
# Instalar librerías
pip install pandas matplotlib

Paso 2: Crear el Script de Simulación

Crea un archivo simulacion_velocidad.py con el siguiente código:

# simulacion_velocidad.py
import pandas as pd
import matplotlib.pyplot as plt

# Datos de entrada: puntos de historia completados por sprint
historias_por_sprint = [5, 3, 8, 2]  # Ejemplo de 4 sprints
puntos_restantes = 60  # Puntos de historia en el backlog
duracion_sprint_dias = 14  # Duración de un sprint en días

# Calcular la velocidad promedio
velocidad = sum(historias_por_sprint) / len(historias_por_sprint)
print(f"Velocidad promedio: {velocidad:.2f} puntos de historia por sprint")

# Proyectar sprints requeridos
sprints_requeridos = puntos_restantes / velocidad
print(f"Sprints requeridos para completar {puntos_restantes} puntos: {sprints_requeridos:.1f}")

# Proyectar días totales
dias_totales = sprints_requeridos * duracion_sprint_dias
print(f"Días estimados: {dias_totales:.0f} días (aproximadamente {dias_totales/7:.1f} semanas)")

# Generar datos para el Burndown Chart
sprints = list(range(1, len(historias_por_sprint) + 1))
puntos_acumulados = [puntos_restantes - sum(historias_por_sprint[:i+1]) for i in range(len(historias_por_sprint))]
puntos_acumulados.insert(0, puntos_restantes)  # Agregar el punto inicial
sprints.insert(0, 0)  # Agregar sprint inicial

# Crear el Burndown Chart
plt.figure(figsize=(10, 6))
plt.plot(sprints, puntos_acumulados, marker='o', linestyle='-', color='b')
plt.title('Burndown Chart Simulado')
plt.xlabel('Número de Sprint')
plt.ylabel('Puntos de Historia Restantes')
plt.grid(True)
plt.savefig('burndown_chart.png')
plt.show()

print("\nGráfico guardado como 'burndown_chart.png'")

Descarga el script:

simulacion_velocidad.py

Paso 3: Ejecutar el Script

Ejecuta el script con:

python simulacion_velocidad.py

Resultado esperado:

Velocidad promedio: 4.50 puntos de historia por sprint
Sprints requeridos para completar 60 puntos: 13.3
Días estimados: 187 días (aproximadamente 26.7 semanas)
Gráfico guardado como 'burndown_chart.png'

El script genera un Burndown Chart mostrando el progreso simulado.
